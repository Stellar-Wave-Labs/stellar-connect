# Security Policy

## Supported Versions

We actively support the following branches with security updates:

| Branch | Supported |
| ------- | --------- |
| `main`  | ✅ Yes |
| `dev`   | ✅ Yes |
| Other branches | ❌ No |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Instead, report it privately by emailing:

**security@example.com**

Include as much information as possible, such as:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Proof of concept (if available)
- Suggested mitigation (optional)

We will:

- Acknowledge receipt of your report as soon as possible.
- Investigate and validate the issue.
- Keep you informed of our progress.
- Work on a fix and coordinate responsible disclosure before publishing details.

We appreciate your efforts to help keep our project and users safe.
